社区公益服务对接管理平台 — 数据库设计说明

数据库名建议：community_service
引擎：MySQL 8.0，字符集 utf8mb4
主 DDL：src/main/resources/db/schema.sql
补丁脚本：同目录 service_request_emergency.sql、sys_region_province_city.sql 等

一、表清单与业务含义

表名说明
sys_region区域/网格树（区→街道→社区）；含省、市展示；parent_id 自关联
sys_user用户；community_id 逻辑关联 sys_region
service_request公益服务需求（发布、审核、状态流转）
service_claim志愿者认领记录及时长
service_evaluation评价（与认领记录一对一）
announcement社区公告
sys_config系统参数 JSON
backup_record备份/导出记录
support_like全局点赞计数
todo_group / todo_task待办分组与任务
audit_log审计日志

三、逻辑关联（未建外键，ER 中建议画出）

来源含义关联目标

todo_task任务归属sys_user.id / todo_group.id
audit_log.user_id操作人sys_user.id（可空）

四、核心业务流程与表协作（文字版）

  1. 居民（sys_user）在 service_request 发布需求。
  2. 社区管理员审核：audit_by_user_id、status、驳回原因等。
  3. 志愿者在 service_claim 认领，完成后填写 service_hours。
  4. 居民在 service_evaluation 对 claim_id 评价（唯一约束：一条认领一条评价）。
  5. 网格展示：community_id 与 sys_region 对应；province / city 仅展示。
